# 文本预处理架构 v1 — 规则候选 + 模型选择

> 核心理念：让规则做规则擅长的（结构化提取、候选生成），让模型做模型擅长的（语义选择、歧义消解）。两者嵌套而非串行。

---

## 一、整体流程

```
原始文本
    │
    ▼
┌─────────────┐
│  文本清洗层   │  Layer 0: 去噪、标准化
└─────────────┘
    │
    ▼
┌─────────────┐
│  结构分析层   │  Layer 1: 分句、句法草图
└─────────────┘
    │
    ▼
┌─────────────┐
│  候选提取层   │  Layer 2: 规则提取主语/宾语/实体候选集
└─────────────┘
    │
    ▼
┌─────────────┐
│  语义选择层   │  Layer 3: 0.5B 从候选集中选择题
└─────────────┘
    │
    ▼
┌─────────────┐
│  片段注入层   │  Layer 4: 主语/宾语贴回对应片段
└─────────────┘
    │
    ▼
┌─────────────┐
│  质量校验层   │  Layer 5: 检查语义完整性、标记低置信度
└─────────────┘
    │
    ▼
  结构化文档
```

## 二、各层详细设计

### Layer 0: 文本清洗层

**职责：** 把原始文本变成「干净的、可分析的句子序列」

**输入：** Markdown / HTML / 纯文本
**输出：** 有序的句子列表，每句带有上下文索引

**规则集：**

```
1. 去除非文本块
   行为: 识别并移除代码块（``` 包裹的内容）、公式（$...$）、表格（|...|行）
   依据: 这些内容的结构规则和正文不同，混入会污染句法分析

2. 清理行内噪声
   行为: 移除 Markdown 标记（**bold**、*italic*、[link]()、# heading）
        将这些标记替换为纯文本内容或空字符串
   理由: 格式化标记不属于语义内容，但保留文本本身

3. 统一标点
   行为: 全半角统一、中文逗号→中文逗号、连续空格→单空格
        中文和英文之间加空格（可选）
   边界情况: 保留引号（用于实体识别）、保留专有名词中的连字符

4. 去重段落
   行为: 识别并移除完全相同或相似度 > 0.95 的连续段落
   依据: 技术笔记中经常出现粘贴重复
```

**置信度打分：** `1.0`（确定性操作，不依赖模型）

---

### Layer 1: 结构分析层

**职责：** 把句子序列变成「带局部上下文的结构化片段」

**输入：** 句子列表
**输出：** 片段列表，每片段附带上下句索引

**规则集：**

```
1. 复合句拆分
   行为: 在逗号、分号、冒号处拆分长句
   约束: 拆出片段不少于 3 个字
         保留「研究表明」、「实验发现」等提示性短语作为前置标记
   理由: 长句内部的因果关系在 RAG 检索时会被稀释

2. 段落边界锚定
   行为: 标记每个片段的段落归属
   输出: fragment.paragraph_id, fragment.sentence_span
   理由: 跨段落的共指消解依赖段落锚点

3. 代词位置标记
   行为: 扫描 "它/他/她/其/该/这/那些/这些/此/本"
   输出: 每个代词在片段中的位置索引 + 类型标记
   类型: [人称代词/指示代词/所有格代词]
   理由: 代词位置是 Layer 2 候选提取的锚点

4. 引号 / 书名号内容提取
   行为: 提取 "" 「」『』《》【】内的内容作为独立语义单元
   输出: entity_candidates.append("提取内容")
   理由: 引号内容是明确的命名实体，无需模型判断
```

**置信度打分：** `0.9~1.0`（语法规则，中文标点高度规范）

---

### Layer 2: 候选提取层

**职责：** 给每个代位词和每个片段，生成「最可能是这个位置主语/宾语的东西」候选列表。

**输入：** 片段序列 + 代词位置标记
**输出：** 每个代位词的候选列表 + 每个片段的候选主语列表

**这是整个架构的核心——规则引擎的价值全在这里。**

```
规则 A: 近端主语候选
  行为: 从代位词所在位置向前搜索，取最近 3 个「名词性短语」
  名词性短语定义:
    专有名词（首次出现+大写/引号/书名号）
    动宾结构中的核心名词（"提出了X" → X 是候选）
    "的"字结构中的核心（"重力知识库的检索效果" → 整体作为候选）
  输出: candidate_list.append((短语, 位置偏移, 类型))

规则 B: 近端宾语候选
  行为: 从代位词所在位置向前向后搜索
        向前取最近 3 个「宾语性短语」
        向后取最近 2 个「动宾结构中的核心」
  标注: 宾语候选需标记「与谓语的关系」（直接宾语/间接宾语/补语）

规则 C: 跨段候选扩展
  行为: 从代位词所在段落向前扩展 2 个段落
        从扩展段落中提取「首次出现」的专有名词/概念
  理由: 宾语代指经常跨段落（第 2 段提到的东西第 5 段还在用）
  边界: 候选总数不超过 7 个（过多会使模型选择退化）

规则 D: 句子主语候选
  行为: 对于无主语的片段（祈使句/省略句），从以下来源提取:
        1. 段落标题 (若有)
        2. 前一句的主语
        3. 所在段落的主题（段落首句的核心名词）
  输出: 每个片段都有至少 1 个主语候选

规则 E: 排除规则（过滤噪声候选）
  行为: 排除以下候选:
        - 长度 < 2 字的非专有名词
        - 纯代词（"我们"/"他们"/"它"）
        - 停用词（"方法"/"问题"/"研究" → 注意这些是高频噪声）
        - 程序代码片段残留
        - 纯数字/日期
  理由: Layer 0 的清洗无法覆盖所有噪声，这里再次过滤

规则 F: 优先级排序
  行为: 候选列表按以下优先级排序:
        P0: 专有名词 + 引号内容（最可信）
        P1: 首次出现的完整概念（"重力知识库" > "知识库"）
        P2: 带修饰语的核心名词（"基于深度学习的模型" > "模型"）
        P3: 带定语的核心名词（"我们提出的方法" → 注意排除代词部分）
        P4: 孤立核心名词（"模型"/"方法"）
        P5: 谓语动词 + 名词（"验证假设" → 动名结构）

规则 G: 宾语具体化（新增，解决宾语被省略问题）
  行为: 检测「验证了」「实现了」「提出了」等谓词后的宾语
        如果宾语缺失或被省略:
        - 向前搜索最近的名词短语
        - 从上下文推断宾语
        - 实在找不到→标记为「需要模型判断」
```

**置信度打分：**

```python
def layer2_confidence(candidate, source_type):
    """
    P0 = 1.0
    P1 = 0.85
    P2 = 0.75
    P3 = 0.60
    P4 = 0.45
    P5 = 0.30
    跨段候选: 基础分 × 0.85（位置越远衰减越多）
    """
```

**边界情况处理：**

| 情况 | 处理 |
|------|------|
| 候选集为空 | 回退至「本句第一个动宾结构」或标记为空 |
| 候选都指向同一事物 | 去重后选最长最完整的 |
| 候选超过 7 个（跨段扩散） | 只保留距离最近且得分最高的 7 个 |
| 规则间候选冲突 | 较高优先级胜出；同优先级 → 位置更近的胜出 |
| 主语候选包含代词 | 排除代词本身，保留其修饰语（"我们的研究" → 排除"我们"） |

### Layer 3: 语义选择层

**职责：** 0.5B 从候选列表中选择「最合理」的那一个。

**输入格式（模板）：**

```
句子: {片段文本}

需要选择的内容: {主语/宾语}
候选列表: [{候选A}, {候选B}, {候选C}]

选择规则:
候选列表中的所有内容。
你的任务是输出合适的选择结果，不要输出其他文字。

选择:
```

**输出约束：** 只能是候选列表中的一个（不允许自创）

**选择和生成的差异对比：**

```python
生成模式（旧）:
  输入: "我们通过实验验证了这一假设"
  输出: "主语|本研究|宾语|深度学习假设"
  错误模式: 格式错乱、词被替换、宾语自创
  错误概率: ~60%（0.5B 实测）

选择题模式（新）:
  输入: "我们通过实验验证了这一假设\n候选: [我们, 本研究, 实验, 该假设]"
  输出: "本研究"
  错误模式: 在候选集中选了不太合适的
  错误概率: ~20%（0.5B 估计，未实测）
```

**选择逻辑（0.5B 的决策路径）：**

```
每选一次，模型实际上在做三件事的判断:
1. 表层匹配: 这个候选在原文中出现过吗？（好选）
2. 语义匹配: 这个候选和当前语境说的是一回事吗？（有点难）
3. 优先级: 有多个匹配时，哪个最「本质、最专有」（最难）

第三点最难——模型需要区分:
  "这个候选是泛称（模型/方法）还是特指（重力知识库）"

可以通过 prompt 中的选择规则来引导。
```

**回退机制：**

```
模型选择的置信度如何判断?
方法: 请求模型时设 options={temperature: 0.1}（低温度）
      多次采样（3次），取频率最高的选择
      
如果 3 次采样完全不一致 → 就认为模型无法判断 → 取候选列表中优先级最高的那个
```

### Layer 4: 片段注入层

**职责：** 将 Layer 3 选出的主语/宾语「贴回」对应的片段。

**注入规则：**

```
每个片段最终呈现的格式:

{
  "fragment_id": "doc_001_par3_sent2",
  "text": "经过实验验证，效果优于传统RAG",
  "explicit_subject": "重力知识库",
  "explicit_object": "传统RAG",
  "relations": [
    {"predicate": "优于", "subject": "重力知识库", "object": "传统RAG"},
    {"predicate": "验证", "subject": "我们", "object": "假设"}
  ],
  "confidence": {
    "subject": 0.85,
    "object": 0.75,
    "relations": [0.90, 0.60]
  }
}
```

**主语注入策略：**

```
策略 A: 片段是完整句且有显式主语 → 不做任何修改
策略 B: 片段是完整句但主语是代词 → 在句首或代词前注入:
         "经过实验验证[重力知识库]，效果优于传统RAG"
策略 C: 片段是省略句（无主语） → 在句首注入:
         "[重力知识库]经过实验验证，效果优于传统RAG"
策略 D: 片段是并列句 → 对每个子句分别判断是否注入
```

**宾语注入策略：**

```
策略 A: 宾语已存在且准确 → 不修改
策略 B: 宾语是代词（"它"/"该"/"此"） → 替换为完整宾语
策略 C: 宾语被省略 → 在谓词后注入:
         "验证了[深度学习模型的泛化能力假设]"
策略 D: 宾语太近（当前句的核心词） → 不注入，保留原文
```

**注入格式：**

```
使用 **方括号 + 斜体** 标记注入内容，区分于原文:
  原文: "经过实验验证，效果优于传统RAG"
  注入后: "[重力知识库]经过实验验证，效果优于[传统RAG]"

存储时保留:
  - original_text: 原始文本
  - enriched_text: 注入后的文本
  - injection_map: [{position: [0,6], content: "重力知识库", type: "subject"}]
```

---

### Layer 5: 质量校验层

**职责：** 检查注入后的片段是否语义通顺，标记需要人工处理的低置信度片段。

**校验规则（纯规则，不依赖模型）：**

```
规则 A: 注入后片段长度异常
  检测: 注入后的片段长度 < 5 字 → 可能切割过短
  处理: 合并相邻片段，重新注入

规则 B: 主语宾语重叠
  检测: explicit_subject == explicit_object
  处理: 保留主语，宾语标记为「需要人工确认」

规则 C: 跨段一致性校验
  检测: 同一概念在不同片段中有不同的 explicit_subject
  行为: 选取出现频次最高的，统一

规则 D: 语义完整性评分
  - explicit_subject 存在且不是代词 → +1 分
  - explicit_object 存在且不是代词 → +1 分
  - 至少有一个 relation → +1 分
  - 没有注入冲突（重叠/自相矛盾） → +1 分
  总分 ≥ 3 → 高质量；总分 = 2 → 中等，可以接受；总分 ≤ 1 → 低质量，建议人工
```

## 三、层间耦合

```
Layer 0 → Layer 1: 干净文本 + 分句
Layer 1 → Layer 2: 片段序列 + 代词位置 + 引号内容
Layer 2 → Layer 3: 候选列表（主语/宾语/跨段）
Layer 3 → Layer 4: 选中的主语/宾语 + 置信度
Layer 4 → Layer 5: 注入后的片段 + injection_map
Layer 5 → 输出: 结构化文档 + 质量报告

每个 Layer 都可以独立测试和替换。
Layer 3 替换为 7B 模型时，其他层不需要改。
```

---

## 四、回退体系

```
假设 0.5B 在 Layer 3 挂了（输出不在候选集中）:

第 1 层回退: 取候选集优先级最高的（规则决定，零成本）
第 2 层回退: 取最后一个非代词的完整主语（所有 Layer 1~2 的通用兜底）
第 3 层回退: 保留原文，标记为主语缺失（不影响下游检索）
第 4 层回退: 整个片段跳过，不处理（保底，0% 幻觉风险）

每一层回退的代价递减，安全递增。
```

## 五、和重力知识库的集成点

```
文本预处理输出 → 重力知识库输入

结构化片段:
  - enriched_text → 用于语义检索（embedding 输入）
  - explicit_subject → 用于角色表构建（RoleTable 的 source）
  - explicit_object → 用于关系提取
  - relations → 用于层级生长（HierarchyGrower 的边）
  - injection_map → 用于检索时还原原文（展示用）

这样:
  Layer 0~5 的输出 ≈ 重力知识库的「预处理食材」
  下游不用关心文本是怎么来的，只关心结构化片段
```

## 六、实现优先级

```
Phase 1（立即可做）:
  Layer 0 + Layer 1 + Layer 2（规则系）
  → 能产生候选列表就算成功
  
Phase 2（接模型）:
  Layer 3（0.5B 选择器）
  → 快速实验，验证选择题模式是否真的比生成模式强
  
Phase 3（全链路）:
  Layer 4 + Layer 5
  → 完整的 text → 结构化片段

Phase 4（验证迭代）:
  用重力知识库的实际数据测试
  统计每层的准确率、回退率、覆盖率
  根据数据调整规则阈值和候选排序
```
