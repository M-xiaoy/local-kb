# 重力知识库 — 一个不走寻常路的本地 RAG 系统（开源）

## 这是个什么项目

一个本地的知识问答系统。上传文档，检索，用 LLM 回答。听起来跟其他 RAG 项目差不多？

区别在于检索层。没用 BM25 混合检索，没用 Cross-encoder reranker，没用 HyDE 查询改写——全部用 **K-Means 聚类 + 向量空间几何运算** 替代了。

## 核心思路

文档入库后自动切片、嵌入、聚类。K-Means 用 Silhouette Score 自动检测最优簇数，不需要手动打标签。

检索走四层：

1. **簇级路由** — 查询向量先检测跟哪个簇最亲和，只搜亲和簇
2. **引力漂移** — 如果初检发现结果集中在某个簇，查询向量向簇质心漂移 15%，等效于 HyDE 但零 token 开销
3. **质量场融合** — 每个球体在入库时自动提取关键词 TF 权重，检索时 0.7 语义分 + 0.3 术语分融合
4. **冗余惩罚** — 同一簇内相似度过高的内容自动降权

## 当前状态

- 存储 1460 个知识球体，自动分为 5 个知识簇
- 全链路检索延迟约 2.1 秒（瓶颈在 Ollama 嵌入模型，检索层本身 ~15ms）
- 支持 Ollama 本地 / DeepSeek API 两种回答生成后端
- 单页 HTML 前端，搜索框居中设计，支持连续会话
- Windows/Linux/macOS 一键部署

## 技术栈

Ollama (nomic-embed-text) + FAISS + sklearn KMeans + FastAPI

开源在 GitHub，纯 Python，无外部 API 依赖。
