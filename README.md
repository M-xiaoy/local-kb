# ✦ 重力知识库 — Gravity Knowledge Base

**一个不走寻常路的本地知识问答系统。**

不是另一份 RAG 教程的复读。这个系统用 **K-Means 聚类自动发现知识场域**，用 **引力漂移替代 HyDE**，用 **质量场替代 BM25 混合检索**——所有改进都建在同一个几何直觉上：知识在向量空间中形成引力场，检索就是在这个场中导航。

## 快速开始

### 依赖

```bash
# 1. 安装 Ollama（如果没装）
curl -fsSL https://ollama.com/install.sh | sh

# 2. 拉取嵌入模型
ollama pull nomic-embed-text

# 3. （可选）拉取回答生成模型
ollama pull qwen2.5:7b
```

### 一键启动

**Windows（PowerShell）：**
```powershell
.\deploy.ps1
```

**Linux / macOS：**
```bash
chmod +x deploy.sh && ./deploy.sh
```

### 手动启动

```bash
# 安装依赖
pip install -r requirements.txt

# 启动服务
python -m uvicorn api.main:app --host 0.0.0.0 --port 8765

# 打开浏览器访问
# http://localhost:8765
```

## 它跟其他知识库有什么不同

### 不靠手动打标签，靠自动聚类

```
传统做法：用户上传文件 → 手动选一个分类（"技术笔记" / "小说创作"）
重力做法：用户上传文件 → K-Means 自动聚类 → 系统自行发现 5 个知识簇
```

每次上传新文件后，全量重聚类自动运行，**Silhouette Score 自动检测最优簇数 K**。用户不需要告诉系统知识应该怎么分类——系统自己看得出来。

### 检索不走标准 RAG，走四层重力排序

```
标准 RAG：   向量检索 Top-100 → Cross-encoder Rerank → Top-5
重力知识库： 簇级路由 → 引力漂移 → FAISS Top-200 → 质量场融合 → 四层排序 → Top-5
```

每一层的设计逻辑：

| 层 | 替代了什么 | 为什么不用标准方案 |
|----|-----------|-------------------|
| **簇级层次索引** | RAPTOR 树搜索 | 簇中心就是天然树根，不额外生成摘要 |
| **引力漂移** | HyDE 查询改写 | 走几何路径修正查询向量，不生成 token |
| **质量场** | BM25 + Dense RRF | 关键词权重写入球体字段，不维护独立索引 |
| **重力冗余惩罚** | Cross-encoder Reranker | 用簇内密度替代二分类推理 |

这些不是为了标新立异而换方案。是因为**重力空间本身提供了足够的基础设施**——簇中心、簇归属、场域亲和度——让标准方案中需要额外模块才能做的事，在同一个几何框架内一步到位。

### 场域路由走 Soft Routing，不硬切

```
传统做法：检测到查询属于"技术笔记" → 只搜技术笔记
重力做法：检测到查询亲和"簇1"(0.69)"簇0"(0.59) → 两个都搜，簇1优先
```

Soft routing 不丢失任何信息——在簇优先排序中属于低亲和度簇的球体仍然出现在候选池中，只是排序靠后。

## 架构总览

```
用户
  ↓
前端界面（搜索 / 聚焦 / 上传 / 会话）
  ↓
知识管线
  文件解析 → 切片 → 嵌入 → 关键词提取 → 球体入库
  ↓
存储层
  ┌─ FAISS 索引（向量粗搜）
  └─ Sphere Store（元数据 + 关键词权重 + 场域 + 簇归属）
  ↓
重力空间检索层
  簇级路由 → 引力漂移 → FAISS 搜索 → 簇优先排序
  → 多样性 + 冗余惩罚 + 场域加权 + 质量场融合
  ↓
  Top-5 球体 + 得分
```

### 核心技术栈

| 组件 | 方案 | 说明 |
|------|------|------|
| 嵌入模型 | `nomic-embed-text`（768d） | Ollama 本地，零 API 费用 |
| 向量索引 | FAISS `IndexFlatIP` | 精确内积搜索（千级够用） |
| 聚类引擎 | `sklearn KMeans` + Silhouette Score | 自动检测最优 K |
| 关键词提取 | `re + Counter`（纯 Python） | 零依赖，球体入库自动计算 |
| 前端 | 单页 HTML（内嵌 FastAPI） | 零前端框架，即开即用 |
| 回答生成 | `qwen2.5:7b`（可选） | Ollama，接上即可 |

## 项目背景

这个项目是[重力空间架构](https://github.com/M-xiaoy/gravity-space-architecture)的第一次工程实践。

重力空间是 2026 年 7 月与小刘共同推演的**个人化模型架构**——思考层不产指令，只维护地形图；行动层不自己决策，只从地形图读信息。这套知识库系统是重力空间在工程上的首次落地尝试，验证了一个想法：**当你在向量空间中有了聚类、场域、质心这些基础设施，很多标准 RAG 中的「额外模块」可以简化为几何运算。**
